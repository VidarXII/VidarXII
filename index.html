Roster Email Parser

The model takes in several batches of emails, parses through them to extract important features such Transaction Type (Add/Update/Term), Transaction Attribute, Complete Address, Phone number, Email ID, and other features that are of immense use in helping hospitals and practitioners sort out their data. The model is a transformer that tokenizes data and uses a non-proprietary LLM (microsoft/Phi-3-mini-4k-instruct) to sort it into an excel file.

To check out the source code, look at my github page!
